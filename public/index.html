Platform

IotEagle Platform

A complete IoT data pipeline — from device to dashboard. Collect sensor data over MQTT, store it in MongoDB, and visualize it in real time.

REST API

Flask-based API for user management, device registration, and data querying with session auth.

MQTT Broker

EMQX 5.x handles all device connections with per-device auth and topic-based ACL rules.

MongoDB

Time-series sensor data stored per user. Each user gets an isolated collection.

Dashboard

React-based read-only dashboard with live charts, device selector, and time range filters.

Architecture

All services run as Docker containers on a single host, connected via host.docker.internal.

Data Flow

  1. An IoT device connects to EMQX using its device_id as username and MQTT password.
  2. The device publishes JSON to its assigned topic: {user_id}/{subtopic}.
  3. mqtt_to_mongo subscribes to #, parses each message, and writes it to MongoDB collection mqtt_data.{user_id} with a device_id and timestamp.
  4. The Dashboard queries main_api which reads from MongoDB and returns the data.

Quick Start

Register a user, add a device, and start sending data in 3 steps.

1

Create an account

curl -X POST https://api.iotlab.am/users/signup \
  -H "Content-Type: application/json" \
  -d '{
    "username": "mydevice",
    "email": "me@example.com",
    "password": "securepass"
  }'
2

Sign in & register a device

# Sign in
SESSION=$(curl -s -X POST https://api.iotlab.am/users/signin \
  -H "Content-Type: application/json" \
  -d '{"username":"mydevice","password":"securepass"}' \
  | jq -r '.session')

# Add device
curl -X POST https://api.iotlab.am/devices/add \
  -H "Content-Type: application/json" \
  -d "{
    \"user_session\": \"$SESSION\",
    \"device_name\": \"My Sensor\",
    \"rules\": [{
      \"action\": \"publish\",
      \"topic\": \"sensors/data\",
      \"permission\": \"allow\"
    }]
  }"
3

Publish sensor data via MQTT

import paho.mqtt.client as mqtt
import json, time

client = mqtt.Client()
client.username_pw_set("DEVICE_ID", "DEVICE_PASSWORD")
client.connect("iotlab.am", 1883)

while True:
    payload = {"temperature": 22.5, "humidity": 60}
    client.publish("{USER_ID}/sensors/data", json.dumps(payload))
    time.sleep(30)

MQTT Integration

Connect any MQTT-capable device to EMQX using the credentials provided by /devices/get.

Connection Details

ProtocolHostPortNotes
MQTT (TCP)iotlab.am1883Standard
MQTT over WSiotlab.am8083WebSocket
MQTT over TLSiotlab.am8883Secure

Authentication

Username: {device_id}       e.g. CEZmobC6SYjT1azK
Password: {device_password} e.g. q56iK4aGiY3POiLm

Topic Structure

{user_id}/{your_topic}

Example: DVONBeK1zitwkwN6/sensors/temperature
Important: Always prefix your topic with your user_id. The device ACL rules already include this prefix — publish to the full topic as returned by /devices/get.

Python Example

import paho.mqtt.client as mqtt
import json, time, random

BROKER   = "iotlab.am"
PORT     = 1883
DEVICE_ID   = "CEZmobC6SYjT1azK"
DEVICE_PASS = "q56iK4aGiY3POiLm"
TOPIC    = "DVONBeK1zitwkwN6/sensors/data"

def on_connect(client, userdata, flags, rc):
    print("Connected" if rc == 0 else f"Failed: {rc}")

client = mqtt.Client()
client.username_pw_set(DEVICE_ID, DEVICE_PASS)
client.on_connect = on_connect
client.connect(BROKER, PORT)
client.loop_start()

while True:
    payload = {
        "temperature": round(20 + random.uniform(0, 10), 2),
        "humidity":    round(50 + random.uniform(0, 30), 2),
    }
    client.publish(TOPIC, json.dumps(payload))
    print(f"Published: {payload}")
    time.sleep(30)

Arduino / ESP32 Example

#include <WiFi.h>
#include <PubSubClient.h>
#include <ArduinoJson.h>

const char* ssid     = "YOUR_WIFI";
const char* password = "YOUR_WIFI_PASS";
const char* mqtt_server   = "iotlab.am";
const char* mqtt_user     = "CEZmobC6SYjT1azK";   // device_id
const char* mqtt_password = "q56iK4aGiY3POiLm";    // device_password
const char* topic         = "DVONBeK1zitwkwN6/sensors/data";

WiFiClient espClient;
PubSubClient client(espClient);

void setup() {
  WiFi.begin(ssid, password);
  while (WiFi.status() != WL_CONNECTED) delay(500);
  client.setServer(mqtt_server, 1883);
}

void loop() {
  if (!client.connected()) {
    client.connect("esp32-sensor", mqtt_user, mqtt_password);
  }

  StaticJsonDocument<128> doc;
  doc["temperature"] = 22.5;   // read from sensor
  doc["humidity"]    = 60.0;

  char buf[128];
  serializeJson(doc, buf);
  client.publish(topic, buf);
  delay(30000);
}

HTTP to MQTT

The http-to-mqtt service acts as a bridge — it accepts a standard HTTP POST request and publishes the payload to the MQTT broker on your behalf. This is useful when your device or application can make HTTP requests but doesn't have a native MQTT library.

No MQTT library needed

Any HTTP client works — curl, Python requests, Arduino HTTPClient, Postman.

Same auth model

Uses the same device_id / password credentials as direct MQTT.

Arduino / ESP32 (HTTPClient)

#include <WiFi.h>
#include <HTTPClient.h>
#include <ArduinoJson.h>

const char* server = "http://iotlab.am:7003/send-to-mqtt";

void sendSensorData(float temp, float hum) {
  HTTPClient http;
  http.begin(server);
  http.addHeader("Content-Type", "application/json");

  StaticJsonDocument<256> doc;
  doc["username"] = "CEZmobC6SYjT1azK";
  doc["password"] = "q56iK4aGiY3POiLm";
  doc["topic"]    = "DVONBeK1zitwkwN6/sensors/data";

  JsonObject data = doc.createNestedObject("data");
  data["temperature"] = temp;
  data["humidity"]    = hum;

  String body;
  serializeJson(doc, body);

  int code = http.POST(body);
  Serial.println(http.getString());
  http.end();
}

void loop() {
  sendSensorData(22.5, 60.0);
  delay(30000);
}
How it works internally: The service creates a fresh MQTT client per request, authenticates with the broker using your device credentials, publishes the JSON payload, then disconnects. The broker's ACL rules still apply — the device can only publish to its allowed topics.

When to use HTTP vs direct MQTT

ScenarioRecommended
Microcontroller with MQTT library (ESP32, Arduino + Ethernet)Direct MQTT
Server-side app or script sending occasional dataHTTP to MQTT
Device behind restrictive firewall (port 1883 blocked)HTTP to MQTT
Rapid prototyping / testing with curl or PostmanHTTP to MQTT
High-frequency streaming (>1 msg/sec)Direct MQTT
Battery-powered device (persistent connection saves power)Direct MQTT

Payload must be valid JSON. Any numeric fields are automatically detected and charted by the dashboard.

Recommended Schema

{
  "temperature": 22.5,

Stored Document

After passing through mqtt_to_mongo, the document stored in MongoDB looks like:

{
  "_id": "ObjectId(...)",
  "device_id": "CEZmobC6SYjT1azK",
  "temperature": 22.5,
  "humidity": 60.1,
  "topic": "DVONBeK1zitwkwN6/sensors/data",
  "timestamp": "ISODate(...)",
  "created_at": "ISODate(...)"
}
Flexible schema: You can send any JSON fields — the dashboard auto-discovers all numeric fields and renders them as charts. No configuration needed.

Dashboard

The dashboard at dashboard.iotlab.am is a read-only public interface — no login required.

Features

  • Device sidebar — lists all registered devices, click to switch
  • Stat cards — total records, in-range count, sensor field count, last data time
  • Latest readings — current value of every numeric sensor field
  • Sensor charts — line charts for all numeric fields, auto-grouped
  • Time range filter — 1H / 6H / 24H / 7D
  • Auto-refresh — updates every 30 seconds automatically
  • Data table — last 50 raw records per device

Rebuilding the Dashboard

If you change credentials or need to rebuild:

cd /root/github/iot-dashboard
DASHBOARD_USER=youruser DASHBOARD_PASS=yourpass \
  docker compose up -d --build

Simulator

The iot-simulator service generates realistic sensor readings every 15 seconds, useful for testing the dashboard without physical hardware.

What it generates

Climate Sensor

  • temperature — 20–25°C, sinusoidal drift
  • humidity — 50–65%, inverse wave

Air Quality Sensor

  • co2 — 350–500 ppm
  • pm25 — 5–15 µg/m³
  • voc — 80–180 ppb

Configuration

Edit docker-compose.yml in /root/github/iot-simulator/:

environment:
  MONGO_URI: mongodb://user:pass@host:27017
  USER_ID: DVONBeK1zitwkwN6          # your user_id
  CLIMATE_DEVICE_ID: CEZmobC6SYjT1azK
  AIR_QUALITY_DEVICE_ID: ALJYrp3KeDaVO7Cl
  INTERVAL: "15"                      # seconds between readings

Commands

# Start
cd /root/github/iot-simulator && docker compose up -d

# View live output
docker logs iot-simulator -f

# Stop
docker compose down
